Keret, csomag, szegmens

A keret → csomag → szegmens → datagram kifejezések a számítógépes hálózatok különböző OSI/TCP-IP rétegeiben használt adat-egységek. A sorrend általában lentről felfelé (fizikai továbbítás felől az alkalmazás felé) értendő.

1. Keret (Frame)

  • A keret az adatkapcsolati réteg (2. réteg az OSI model-ben) adat-egysége.
  • A hálózat helyi szakaszán (pl. egy LAN-on) így továbbítják az adatot.
  • Tartalmaz:
    • forrás MAC-cím
    • cél MAC-cím
    • hibadetektáló mező (pl. CRC)
    • benne egy csomag.

Példa: egy Ethernet keret.

👉 Egyszerűen: a helyi hálózaton mozgó „doboz”, ami az IP-csomagot viszi.

2. Csomag (Packet)

  • A csomag a hálózati réteg (3. réteg) adat-egysége.
  • Leggyakrabban IP csomag.
  • Tartalmaz:
    • forrás IP-cím
    • cél IP-cím
    • benne egy szegmens vagy datagram.

Tipikus protokoll:

  • Internet Protocol (IP)

👉 Ez az az egység, ami routereken keresztül az interneten utazik.

3. Szegmens (Segment)

  • A szegmens a szállítási réteg adat-egysége TCP használatakor.
  • A protokoll:
    • Transmission Control Protocol (TCP)
  • Jellemzők:
    • sorszám
    • visszaigazolás
    • hibajavítás
    • megbízható adatátvitel.

👉 Ez a megbízható kapcsolat-orientált kommunikáció adatblokkját jelenti.

4. Datagram

  • A datagram szintén a szállítási réteg adat-egysége, de UDP esetén.
  • A protokoll:
    • User Datagram Protocol (UDP)
  • Jellemzők:
    • nincs kapcsolatfelépítés
    • nincs garantált kézbesítés
    • gyorsabb, kevesebb overhead.

👉 Példák: online játékok, streaming, DNS.

Egymásba ágyazva (encapsulation):

Keret
 └─ Csomag (IP packet)
     └─ Szegmens (TCP) / Datagram (UDP)
         └─ Alkalmazási adat

Röviden:

Fogalom Réteg Példa
Keret adatkapcsolati Ethernet
Csomag hálózati IP
Szegmens szállítási TCP
Datagram szállítási UDP
