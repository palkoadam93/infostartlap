1. A parancssoros konfiguráció lehetőségei

A kapcsoló egy olyan eszköz, amely az egyik portján beérkező üzenetfolyamot egy másik portjára irányítja a célszámítógép MAC-címe alapján. A kapcsoló nem képes a két különböző helyi hálózat között zajló forgalom irányítására. A kapcsoló az OSI-modell második rétegében működik. A második réteg az adatkapcsolati réteg. A Cisco kapcsolók Ip alapú felügyeleti szoftverrel vagy telnet-kapcsolaton keresztül történő kezeléséhez egy felügyeleti IP cím beállítása szükséges. Amennyiben a kapcsoló nem rendelkezik IP-címmel, a konfigurációs feladatok végrehajtásához közvetlenül a konzolportjára kell csatlakozni, és terminálemulációs programot kell használni. Az állomásnév és a jelszavak beállítása ugyanazokkal a parancsokkal történik, mint az ISR esetében. A Cisco kapcsolók IP-alapú felügyeleti szoftverrel vagy telnet-kapcsolaton keresztül történő kezeléséhez állítsunk be egy felügyeleti IP-címet! Ahhoz, hogy egy IP-cím a kapcsolóhoz társítható legyen, a címet egy virtuális helyi hálózati (VLAN) interfészhez kell rendelnünk. A VLAN lehetővé teszi egynél több fizikai port egyetlen logikai csoportként történő kezelését. A kapcsolón egy VLAN van előre konfigurálva, a felügyeleti feladatokat ellátó VLAN1. A VLAN1 felügyeleti interfészéhez tartozó IP-cím beállításához lépjünk be a globális konfigurációs módba!

VLAN1 menedzsment IP beállítása

Kapcsoló alap IP konfigurálása (VLAN1)

Belépés privilegizált és globális konfigurációs módba

Switch> enable Switch# configure terminal

VLAN1 interfész konfigurálása

Lépjünk be a VLAN1 interfész konfigurációs módjába, majd adjuk meg a felügyeleti IP-címet.

Switch(config)# interface vlan 1 Switch(config-if)# ip address 192.168.1.2 255.255.255.0 Switch(config-if)# exit Switch(config)# ip default-gateway 192.168.1.1 Switch(config)# end

Konfiguráció mentése

Switch# copy running-config startup-config

Az IP-címnek érvényesnek kell lennie a helyi hálózatban. A default gateway a közvetlenül csatlakoztatott router interfészének címe.

Kapcsoló összekötése a forgalomirányítóval

Használjunk egyenes kötésű kábelt. A LED-ek jelzik a fizikai kapcsolatot.

IP-beállítások ellenőrzése

Switch# show running-config

Kapcsolat tesztelése

Switch# ping 192.168.1.1

Ezután a routerről pingeljük meg a kapcsoló VLAN1 IP-címét. Ha nem sikeres, ellenőrizzük a kábelezést és az IP-beállításokat.

Portbiztonság (Port Security)

Statikus MAC-cím megadása

S1# configure terminal S1(config)# interface fastEthernet 0/18 S1(config-if)# switchport mode access S1(config-if)# switchport port-security mac-address 0001.0002.0003

Dinamikus MAC tanulás

S1# configure terminal S1(config)# interface fastEthernet 0/18 S1(config-if)# switchport mode access S1(config-if)# switchport port-security S1(config-if)# end

A dinamikusan tanult MAC-címek újraindítás után elvesznek. Alapértelmezett maximum: 1 MAC-cím / port.

Sticky MAC tanulás

S1# configure terminal S1(config)# interface fastEthernet 0/18 S1(config-if)# switchport mode access S1(config-if)# switchport port-security S1(config-if)# switchport port-security maximum 50 S1(config-if)# switchport port-security mac-address sticky S1(config-if)# end

Sticky módban a megtanult címek bekerülnek az aktív konfigurációba. Biztonságsértés esetén a port alapértelmezetten shutdown állapotba kerül.

VTP ellenőrzése

Switch# show vtp status Switch# show vtp password

VTP Pruning

Switch(config)# vtp pruning

A VTP lehetővé teszi a VLAN információk automatikus szinkronizálását a trönk kapcsolatokon keresztül. Server módban VLAN létrehozható és terjeszthető, Client módban csak fogadja a módosításokat, Transparent módban nem módosítja a VTP adatbázist, de továbbítja a VTP kereteket. A pruning csökkenti a fölösleges VLAN forgalmat a trönkökön.
